# Pixel Life ‚Äî Godot 4 Life Simulator: Build Plan

## Context

**Greenfield project.** Working directory `C:\Users\wysoc\Desktop\Pixel Life` is empty. The user wants a top-down pixel-art life simulator inspired by Stardew Valley + The Sims: walk a town, hold a job, earn money, buy a house, date and marry an NPC, raise kids, age and die, then play as your adult child to continue the lineage.

**Why this plan exists**: it will be handed to autonomous coding agents (Nessie) to implement. Every milestone, file path, signal name, and Resource schema is fixed up front so agents can pick up any milestone without ambiguity.

**Locked decisions** (from clarification with user):
- Engine: **Godot 4.x, GDScript**
- Art: **Free asset packs** (LimeZu Modern Interiors + Exteriors as primary, Kenney/OGA fallback)
- Save: **Single auto-save slot**
- Time: **Stardew-style** ‚Äî ~15 real min per in-game day, seasons, years, aging, death
- Relationships: **Sims-lite** ‚Äî date ‚Üí marry ‚Üí kids ‚Üí playable child after death
- World: **Town + Farm + Forest + Beach** (4 connected zones)
- Careers: **All four categories** ‚Äî Service, Skilled, Self-employed, Creative

---

## Top-level technical decisions

| Decision | Choice | Why |
|---|---|---|
| Tile size | **16√ó16** | Matches LimeZu, Kenney, most OGA tilesets |
| Display | 320√ó180 base, integer scale 4x | Crisp pixels, fits 1080p natively |
| Save format | **Custom Godot Resource (.tres) + JSON sidecar** | Resource = typed/refactor-safe; JSON = debug + recovery |
| Scene loading | **Manual zone swap under SceneRouter** with persistent player + UI in autoloads | Frees memory between zones; HUD/player survive transitions |
| Data | **`resources/` defines classes, `data/` holds .tres content** | Adding new items = new .tres, no script changes |

## Autoloads (14 singletons)

Register in `Project Settings ‚Üí Autoload` in this order (load-order matters):

1. **`EventBus`** ‚Äî signal hub. No methods, only signals (see ¬ßSignals).
2. **`GameState`** ‚Äî holds active `CharacterProfile`, current zone, pause flag. Methods: `start_new_game`, `switch_active_character` (succession), `pause/resume`.
3. **`TimeManager`** ‚Äî drives clock. Constants: `MINUTES_PER_DAY=1440`, `DAYS_PER_SEASON=28`, `REAL_SECONDS_PER_GAME_MINUTE=0.625`. Methods: `pause_time/resume_time/set_time_scale/advance_to_next_morning/serialize/deserialize`.
4. **`SaveManager`** ‚Äî reads/writes `SavePayload`. `SAVE_PATH := "user://saves/slot0.tres"`, JSON sidecar at `slot0.json`. Methods: `save_now(reason)`, `load_slot()`, `has_save()`, `delete_save()`. Auto-saves on sleep, on quit (`NOTIFICATION_WM_CLOSE_REQUEST`), and on player death.
5. **`EconomyManager`** ‚Äî money, transactions. `get_money/add_money/try_spend/price_of/sell_value`.
6. **`NeedsManager`** ‚Äî Energy, Hunger, Hygiene, Social, Fun (0-100). Decays per `hour_passed`. `get/modify/apply_action_effects`.
7. **`InventoryManager`** ‚Äî 6√ó4 slot grid of `ItemStack`. `add/remove/count/move_slot/get_slot`.
8. **`RelationshipManager`** ‚Äî `npc_id ‚Üí {value, stage, history}`. Stages: `stranger, acquaintance, friend, close_friend, dating, engaged, married, ex, widowed`. `modify/get_stage/try_advance_stage/propose_marriage`.
9. **`JobManager`** ‚Äî `current_job, career_level, performance_avg, fame`. `apply_for/start_shift/end_shift/try_promote/quit`.
10. **`FamilyManager`** ‚Äî aging, pregnancy, succession. Constants: `LIFE_EXPECTANCY_MIN=70`, `LIFE_EXPECTANCY_MAX=90`, `PREGNANCY_DAYS=14`, `STAGE_AGE={baby:0, child:5, teen:13, adult:18}`. `try_conceive/tick_year/age_actor/try_die/move_out_adult_child/succeed_to`.
11. **`DialogueManager`** ‚Äî runs `DialogueData` trees. Pauses time during dialogue. `start/choose/end`.
12. **`AudioManager`** ‚Äî music bus per zone (day/night variants), SFX pool. Crossfades on `zone_changed`.
13. **`SceneRouter`** ‚Äî owns the persistent UI CanvasLayer and player node. `goto_zone(zone_id, spawn_marker)`: fade ‚Üí free old zone ‚Üí instance new ‚Üí reparent player at marker ‚Üí fade in ‚Üí emit `zone_changed`.
14. **`InputActions`** ‚Äî input action name constants (`MOVE_UP`, `INTERACT`, etc.) to prevent typos.

## Resource classes (in `resources/`)

All `extends Resource` with `class_name`:

- **`ItemData`** ‚Äî id, name, description, icon, category, stack_size, base_buy_price, base_sell_price, tags[], effects{}
- **`CropData`** ‚Äî id, seed_item, harvest_item, growth_stages, days_per_stage, seasons[], regrow, regrow_days
- **`JobData`** ‚Äî id, name, category, shift_hours (Vector2i), shift_days[], base_pay_per_shift, requires_education, career_track, resolution ("auto"|"minigame"|"skill_check"), minigame_scene
- **`CareerTrackData`** ‚Äî levels[] of `{title, pay, days_required, performance_required}`
- **`NPCData`** ‚Äî id, name, gender, birthday, home_id, work_job_id, sprite_set, personality_tags[], loved_gifts[], liked_gifts[], disliked_gifts[], hated_gifts[], dialogue_id, schedule_id, marriageable, compatibility{}
- **`DialogueData`** ‚Äî nodes{} keyed by node_id, each `{text, choices[]}`
- **`ScheduleData`** ‚Äî entries[] of `{hour, minute, zone_id, target_marker, action}`, seasonal_overrides{}, weekly_overrides{}
- **`HouseTierData`** ‚Äî tier, name, cost, interior_scene, furniture_slots, bed_count
- **`FurnitureData`** ‚Äî id, item, placed_scene, footprint, provides_action
- **`RecipeData`** ‚Äî id, inputs[], outputs[], station, time_minutes
- **`RelationshipStageData`** ‚Äî stage, min_value, max_value, unlocks[]
- **`AudioTrackData`** ‚Äî id, stream, loop, volume_db, bus
- **`CharacterProfile`** ‚Äî id, name, gender, portrait_index, sprite_index, birth_year, age, parents[], spouse, children[], money, inventory_data (sub-Resource), needs_data (sub-Resource), relationships{}, job_state, house_tier, current_zone, last_position
- **`FamilyTree`** ‚Äî members[] (all CharacterProfiles, living + deceased + kids), active_id, deceased_ids[]

## Save schema (`SavePayload` Resource)

```
SavePayload
‚îú‚îÄ‚îÄ version: int                      # bump on schema changes; SAVE_VERSION = 1
‚îú‚îÄ‚îÄ created_at_unix: int
‚îú‚îÄ‚îÄ time_state: Dictionary            # TimeManager.serialize()
‚îú‚îÄ‚îÄ family_tree: FamilyTree           # spans generations
‚îú‚îÄ‚îÄ active_character_id: String
‚îú‚îÄ‚îÄ world_state:
‚îÇ   ‚îú‚îÄ‚îÄ zone_states { zone_id: { crops_state, foraged_today, chopped_trees } }
‚îÇ   ‚îú‚îÄ‚îÄ npc_overrides { npc_id: { current_zone, last_seen_position, schedule_offset } }
‚îÇ   ‚îú‚îÄ‚îÄ house_tier: int
‚îÇ   ‚îî‚îÄ‚îÄ owned_furniture[]
‚îú‚îÄ‚îÄ economy_state { money, transaction_log_tail[] }
‚îú‚îÄ‚îÄ unlocks { recipes[], areas[], achievements[] }
‚îî‚îÄ‚îÄ settings { music_vol, sfx_vol, ... }
```

**Auto-save triggers**: `sleep_completed`, `NOTIFICATION_WM_CLOSE_REQUEST`, `player_died` (before succession prompt).

**Multi-generation succession**: `FamilyTree.members` keeps every character (living and dead) with `deceased` + `death_year` flags. On death ‚Üí save ‚Üí prompt user to pick eligible adult child ‚Üí `GameState.switch_active_character(child)`. **Inheritance**: house passes to spouse if alive else eldest adult child; money 50% to spouse, 50% split among children; relationships are per-character (not transferred); job/career resets. NPCs can have memory of deceased parent surfaced in dialogue.

After 2 generations of distance, deceased characters retain only summary fields (name, lifespan, key events) to cap save bloat.

Implement `Player` (CharacterBody2D, AnimatedSprite2D, 4-dir idle/walk, run on Shift). `SceneRouter` instances player into Town's YSort. Camera2D bounded to map.

### M10 ‚Äî Housing tiers + build/buy

### M11 ‚Äî Family: marriage, pregnancy, kids, aging

Spouse pathfinds to player house, schedule overridden to home zone. `FamilyManager` aging (`year_passed` listener), children age baby ‚Üí child ‚Üí teen ‚Üí adult. ### M12 ‚Äî Death, succession, inheritance

On death: save ‚Üí succession UI listing eligible adult children ‚Üí `GameState.switch_active_character`. Inheritance rules centralized in `scripts/systems/save/inheritance_rules.gd`. Adult-child move-out option (their own tier-1 apartment). "No heirs" ‚Üí restart with preserved town or new world.
**Test**: speedrun via debug `time_scale=600` to hit death; verify successor inherits house + money; play 2 generations.

### M13 ‚Äî Polish (audio, animations, content pass)
`AudioManager` with per-zone music + day/night variants. SFX for footsteps (per surface), interact, UI, dialogue blip. Particle FX for sleep, level up, marriage. Expand to 8-12 NPCs with full dialogue + romance arcs. Balance pass on needs decay, prices, careers.

## Risks & key tradeoffs

1. **NPC continuity across zones** ‚Äî NPC at the bakery is "off-screen" while player is at the farm. Solution: simulate schedules in pure data via `NPCScheduleRunner`; instantiate the visible sprite only when player enters the NPC's current zone. `npc_overrides` in save reflects last-known state.

2. **Resource save schema lock-in** ‚Äî renaming `character_profile.gd` after M6 breaks all loads. Mitigation: pin Resource scripts to stable paths under `resources/`; never move them; maintain `SAVE_VERSION` and a migration table; JSON sidecar provides recovery.

3. **NPC schedule content treadmill** ‚Äî Stardew schedules with festivals/weather/relationship variants is endless work. Mitigation: ship M11 with 3 NPCs and one base schedule each. Add `seasonal_overrides` only after M11 stabilizes. Defer festivals.

4. **Mini-game scope creep** ‚Äî one mini-game per career *category* maximum: register-rush (service skilled tier), painting timing-bar (creative), fishing minigame (self-employed). Skill check is the default; mini-games are upgrades.

5. **Multi-generation save bloat** ‚Äî cap historical detail; deceased characters >2 generations old keep only summary fields. Centralize inheritance logic in one file for testability.

6. **Pixel-art style consistency** ‚Äî pick *one* style family at M1 (recommended: LimeZu) and stick with it. Mixing Kenney 1-bit + LimeZu + LPC will look broken.
